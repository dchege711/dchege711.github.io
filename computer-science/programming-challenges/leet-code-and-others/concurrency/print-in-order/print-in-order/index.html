Print in Order

Dated Jun 1, 2026; last modified on Mon, 01 Jun 2026

Suppose we have a class:

public class Foo {
  public void first() { print("first"); }
  public void second() { print("second"); }
  public void third() { print("third"); }
}

… and the same instance of Foo is passed to three different threads. Thread A will call first(), thread B will call second() and thread C will call third(). Design Foo such that second() is executed after first(), and third() is executed after second().

See Thread-Based Parallelism - Python for a quick primer on Python’s synchronization primitives.

import threading

from typing import Callable

class Foo:
    def __init__(self):
        self._first_is_done = threading.Event()
        self._second_is_done = threading.Event()

    def first(self, printFirst: 'Callable[[], None]') -> None:
        printFirst()
        self._first_is_done.set()

    def second(self, printSecond: 'Callable[[], None]') -> None:
        self._first_is_done.wait()
        printSecond()
        self._second_is_done.set()

    def third(self, printThird: 'Callable[[], None]') -> None:
        self._second_is_done.wait()
        printThird()
