Bloated Dependencies A bloated dependency is one which is packaged in the application binary, but is not needed to run the application. There are two levels to this: (1) a source file declares a dependency on foo but never actually uses foo, and (2) the application as a whole never uses foo. An optimal de-bloating solution would first address (1) and then tackle (2). Some languages may have better tooling than others when it comes to automatically de-bloating their dependencies....
