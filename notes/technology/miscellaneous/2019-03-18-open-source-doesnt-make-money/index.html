Open Source Doesn’t Make Money Because It Isn’t Designed To Make Money [Ian Bicking]

Dated Mar 18, 2019; last modified on Sat, 01 Feb 2020

Open Source Doesn’t Make Money Because It Isn’t Designed To Make Money. Ian Bicking. http://www.ianbicking.org/blog/2019/03/open-source-doesnt-make-money-by-design.html. 2019-03-18.

A [fallacious] Just World hypothesis: if something has utility, if it helps people, if it’s something the world needs, if it empowers other people, then there should be a revenue opportunity.

Classical scratch-your-own-itch open source software is built by programmers for programmers. And it is wildly successful, but it’s selling to people who aren’t willing to pay.

But programmers are relatively high paid. And they know that good software costs money. Still, OSS is worn as a sign of virtue. Enterprise code - the majority of software out there - affects more people but doesn't command the same number of brownie points.

So much software is open source except any commercial product. Software that people actually touch isn’t free or open.
